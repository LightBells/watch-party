# Watch Party - ローカル動作確認手順

## 必要な準備

1. **Node.js のインストール**
   - Node.js v18以上が必要です
   - https://nodejs.org/ からダウンロード

2. **プロジェクトのセットアップ**
   ```bash
   # 依存関係のインストール（既に実行済み）
   npm install
   ```

## サーバーの起動

1. **TypeScript のビルド**
   ```bash
   npm run build:server
   ```
   
   - もしくは `npm run build` でサーバーと拡張機能をまとめてビルドできます。

2. **開発サーバーを起動**
   ```bash
   npm start
   ```
   
   - 開発中はホットリロード対応の `npm run dev` (ts-node-dev) も利用できます。

3. **サーバーの動作確認**
   - ブラウザで `http://localhost:3000` にアクセス
   - 「Watch Party - テスト用ページ」が表示されることを確認
   - 「サーバーに接続されています」と表示されることを確認

## Chrome拡張機能の読み込み

1. **ビルド済みスクリプトの生成**
   ```bash
   npm run build:extension
   ```
   - 出力先は `extension/dist/` です。開発中は `npm run build:extension -- --watch` を使うと便利です。

2. **Chrome拡張機能の開発者モードを有効化**
   - Chromeブラウザで `chrome://extensions/` にアクセス
   - 右上の「デベロッパーモード」をONにする

3. **拡張機能を読み込み**
   - 「パッケージ化されていない拡張機能を読み込む」をクリック
   - プロジェクトの `extension/dist` フォルダを選択
   - 「Watch Party」拡張機能が一覧に表示されることを確認

4. **拡張機能アイコンの確認**
   - ブラウザのツールバーに「Watch Party」のアイコンが表示される
   - アイコンが表示されない場合は、パズルピースアイコンから固定する

## 動作確認手順

### 1. 基本的な接続確認

1. **テストページにアクセス**
   - `http://localhost:3000` を開く
   - 拡張機能のアイコンをクリック

2. **ルームの作成**
   - 拡張機能ポップアップで「新しいルームを作成」をクリック
   - ルームIDが自動生成される
   - 「ルーム XXXXXX に接続中」と表示されることを確認

### 2. 複数ユーザーでの同期確認

1. **別のブラウザタブを開く**
   - 新しいタブで `http://localhost:3000` にアクセス
   - 拡張機能のアイコンをクリック

2. **同じルームに参加**
   - 最初のタブで作成されたルームIDを入力
   - 「ルームに参加」をクリック
   - 両方のタブで「参加者」リストが更新されることを確認

3. **動画同期のテスト**
   - 一方のタブで動画を再生/停止
   - もう一方のタブでも同じ動作が自動実行されることを確認
   - シークバーを操作して、時間同期も確認

4. **チャット機能のテスト**
   - 拡張機能ポップアップでメッセージを送信
   - 動画画面上にコメントが表示されることを確認
   - 他のタブでもコメントが表示されることを確認

## トラブルシューティング

### サーバーが起動しない場合

1. **ポート3000が使用されている**
   ```bash
   # 他のプロセスでポート3000を使用している場合
   lsof -ti:3000 | xargs kill -9
   ```

2. **依存関係のエラー**
   ```bash
   # node_modulesを削除して再インストール
   rm -rf node_modules package-lock.json
   npm install
   ```

### 拡張機能が動作しない場合

1. **拡張機能の再読み込み**
   - `chrome://extensions/` でリロードボタンをクリック
   - ブラウザタブもリロード

2. **Console エラーの確認**
   - F12で開発者ツールを開く
   - Console タブでエラーメッセージを確認

3. **Socket.IO接続エラー**
   - サーバーが起動していることを確認
   - `http://localhost:3000/socket.io/` にアクセスしてSocket.IOが動作していることを確認

### 動画同期が効かない場合

1. **動画要素の検出**
   - 開発者ツールのConsoleで「Video element found」が表示されることを確認
   - 表示されない場合は、動画要素のセレクターを確認

2. **WebSocket接続状態**
   - 拡張機能ポップアップで接続状態を確認
   - 「接続中」と表示されていることを確認

## 実際の動画サービスでのテスト

### Prime Video での確認

1. **Prime Video にアクセス**
   - https://www.amazon.co.jp/gp/video/ にアクセス
   - 動画を再生

2. **拡張機能の動作確認**
   - 右上に「Watch Party」のステータスが表示されることを確認
   - 拡張機能でルームを作成・参加

### dアニメストア での確認

1. **dアニメストア にアクセス**
   - https://animestore.docomo.ne.jp/ にアクセス
   - 動画を再生

2. **拡張機能の動作確認**
   - 拡張機能が正常に動作することを確認

- **CORS設定**: 本番環境では適切なCORS設定が必要です
- **HTTPS**: 実際の動画サービスではHTTPS接続が必要です
