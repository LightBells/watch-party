# UI構造変更仕様

**実装日**: 2025-01-05  
**バージョン**: 2.0  
**最終更新**: 2025-01-05

## 概要
Watch Party ExtensionのUIを大幅に再設計し、ユーザビリティと機能の分離を改善。

## 変更内容

### 1. Extension Popup（拡張機能ポップアップ）
**変更前**: ユーザーネーム + ルーム管理 + チャット  
**変更後**: ユーザーネーム設定のみ

#### 新機能
- シンプルなユーザーネーム設定専用UI
- 文字数カウンター（20文字制限）
- リアルタイム保存状態フィードバック
- 既存タブのユーザーネーム自動更新

### 2. フローティングボタン（動画ページ右上）
**新規追加**: クリック可能なステータス表示ボタン

#### 機能
- 現在の接続状態とルームIDを表示
- クリックでルーム管理ポップアップを表示
- ホバーアニメーション
- フルスクリーン対応

### 3. ルーム管理ポップアップ
**新規追加**: フローティングボタンから起動されるポップアップ

#### 機能
- ルーム参加・作成
- 接続状態インジケーター
- メンバーリスト表示
- ルーム退出
- 外部クリックで自動閉じ

### 4. コメント入力エリア（画面右下）
**新規追加**: 右側から展開される折りたたみ可能なコメント入力UI

#### 機能
- 画面右下に配置された矢印アイコンボタン（"‹" / "›"）
- 右側から滑り出すパネルデザイン
- ホバー・オープン時のスライドアニメーション
- Enterキーで送信
- フルスクリーン時の位置調整
- 半透明背景でコンテンツを遮らない
- ミニマルなデザイン（テキストなし、アイコンのみ）

## UI要素詳細

### フローティングボタン
```css
位置: 固定位置（右上20px）
サイズ: 自動調整
背景: 半透明黒（backdrop-filter対応）
アニメーション: ホバー時の浮上効果
```

### ルーム管理ポップアップ
```css
位置: フローティングボタン下（70px）
サイズ: 320px幅（モバイル時は画面幅-40px）
背景: 白色、丸角16px
シャドウ: 深めのドロップシャドウ
アニメーション: スライドイン効果
```

### コメント入力エリア
```css
位置: 画面右下固定（bottom: 20px, right: 20px）
背景: 半透明白（backdrop-filter対応）
トグルボタン: 50x50px円形、左丸角デザイン（右側から生える効果）
アイコン: "‹"（閉じた状態）/ "›"（開いた状態）
入力フォーム: 最大幅400px、右寄せ配置
アニメーション: 左スライド展開（translateX）
```

## データフロー

### ユーザーネーム管理
1. Extension Popup → Chrome Storage（globalUsername）
2. Chrome Storage → Content Script（自動同期）
3. Content Script → Server（ルーム参加時）

### ルーム管理
1. Content Script → Server（API呼び出し）
2. Server → Content Script（WebSocket）
3. Content Script → UI更新

### コメント送信
1. Content Script → Server（WebSocket）
2. Server → 全参加者（WebSocket）
3. Content Script → 画面表示

## レスポンシブ対応

### モバイル（768px以下）
- ポップアップ幅を画面幅に調整
- フローティングボタンの余白調整
- コメント入力フォームの余白調整

### フルスクリーン対応
- Prime Video: top位置を80px調整
- dアニメストア: top位置を80px調整  
- コメント入力: bottom位置を60px調整

## アクセシビリティ

### キーボード操作
- Enterキーでのコメント送信
- Escキーでのポップアップ閉じ（今後実装予定）

### フォーカス管理
- コメントパネル展開時の自動フォーカス
- タブキーでの要素移動

## アニメーション

### フローティングボタン
- ホバー時: `translateY(-2px)` + シャドウ強化
- トランジション: `0.2s ease`

### ポップアップ
- 表示時: スライドイン（`translateY(-10px)` → `0`）
- 非表示時: `display: none`（瞬間）

### コメントパネル
- 展開時: 左スライド（`translateX(20px)` → `0`）
- トグルボタン: ホバー・オープン時左スライド（`translateX(-8px)`）
- オープン時の色変化: 黒 → 青（アクティブ状態表示）

### CSS設計
- BEM風命名規則（`wp-` プレフィックス）
- CSS Grid/Flexboxレイアウト
- CSS Custom Properties（今後実装予定）

## 互換性

### 既存機能との互換性
- 既存のWebSocket通信は変更なし
- サーバーAPIは変更なし
- ユーザーネーム機能は完全互換

### ブラウザ対応
- Chrome/Edge: 完全対応

- Safari: 基本機能対応

## トラブルシューティング

### 一般的な問題
1. **フローティングボタンが表示されない**
   - 動画要素の検出失敗
   - z-indexの競合

2. **ポップアップが正常に動作しない**
   - 他のサイトのCSS干渉
   - イベントリスナーの重複登録

3. **コメント入力が反応しない**
   - WebSocket接続の確認
   - ユーザーネーム設定の確認

### デバッグ方法
```javascript
// デバッグモードの有効化
chrome.storage.local.set({ debugMode: true });

// コンソールでの状態確認
document.querySelector('#wp-floating-button');
document.querySelector('#wp-room-popup');
```
